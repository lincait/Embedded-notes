# Linux、脚本

- Linux系统一切皆文件，所有东西都是文件的形式，并且都可以以文本形式打开。
- `sudo`就是把后面跟的命令交给`root`执行，并且只有在`root`组里的用户才能用`sudo`。
- 文件夹权限常使用`chmod 755`，即组和其他人只能读和执行。
- 每个用户必须属于一个**主组**，创建用户时就会自动为其创建一个和用户同名的主组。除此之外，用户还可以加入其他组，这些额外的组为**附加组**。
- 一般情况下，用户创建的文件或目录，其所述组默认被分配到创建者的主组。
- 使用`tar`压缩文件时，直接用`-zcvf`，解压用`-zxvf`。
- 使用`bash`或`sh`命令执行文件时，不需要文件有执行权限。
- `windows`系统下的换行符是`CRLF`，`unix`系统下的换行符是`LF`，文件传输时需要注意这个。
- `gcc`常用选项：`-c`编译不链接，`-g`生成调试信息，`-Wall`开启常用警告。
- `gcc`在执行命令编译时，不需要指定.h文件，它会根据.c中引用的.h自己去找。
- `gcc`在链接库时默认用动态链接，使用`-lxx`选项链接时，链接的是什么库就用什么链接。

# 系统编程

- 系统调用：普通的C函数全程在用户态执行，系统调用（由操作系统内核提供的API）会切换到内核执行。
