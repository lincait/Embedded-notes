# 顺序表

## 形式

用一组地址连续的存储单元，依次存放线性表中的数据元素。存放的数据按顺序一个挨着一个。

## 优缺点

只要知道下标，就能立即访问对应元素。

但在指定位置操作时，需要大片移动后续元素。

## 实现

```c
#define MAXSIZE 100
typedef struct {
    int data[MAXSIZE];
    int length;    // 当前长度
} SeqList;
```



利用当前长度和元素的一一对应来进行增删改查。

# 单向链表

## 形式和作用

由一系列节点组成，每个节点包含数据域和指针域，数据域存放当前节点数据，指针域指向下一个节点。

链表在内存中不要求连续存储，增删高效。

单向链表随机访问时只能按顺序一个个访问，并且要访问前面的只能从头再来

## 实现

### 定义

```c
typedef struct Node {
    int data;           // 数据域
    struct Node* next;  // 指针域，指向下一个节点
} Node;
```

### 基本功能

```c
void func(Node *cur)		
{
    Node *newnode = malloc(sizeof(Node));	//创建新节点
    
    cur = cur->next;		//向后移动，遍历
    
    newnode->next = cur->next;	//在cur后面插入新节点newnode
    cur->next = newnode;
    
    Node *tmp = cur->next;	//删除cur的后继节点
    cur->next = tmp->next;
    free(tmp);
}
```

### 有头和无头

有头：首地址`head`指向头节点，`head->next`才指向第一个有效节点。

无头：首地址`head`直接指向第一个有效节点。

由上述区别导致两种链表在操作实现上有所不同，尤其在边界。注意无头的把`head`传入函数时，要在函数返回值把地址返回或用二级指针，因为函数操作都只给了形参。
