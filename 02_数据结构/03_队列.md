# 相关概念

队列的数据**先进先出**，用队首和队尾来控制队列。

# 顺序存储队列

入队时队尾索引后移，出队时队首索引后移。

## 普通队列

判空条件：`head == tail`

判满条件：`tail == MAXSIZE -1`

实际中，普通队列在不断入队出队的操作下，首尾索引一直在向后移动，这就会导致前面已经出队的位置空着用不了，会浪费大量资源，即”假溢出“。

## 循环队列

### 定义

```C
#define MAXSIZE 100
typedef struct
{
    int data[MAXSIZE];
    int head;	// 队首索引
    int tail;	// 队尾索引
}queue;
```

### 实现

为了区分空和满，需要浪费一个存储单元。规定队首`head`的位置永远不存放数据。

`(sq->tail + 1) % MAXSIZE`：`tail+1`到`MAXSIZE`的时候，取模后刚好是0，回到原点；`tail+1`没到`MAXSIZE`的时候，取模还是自己，正常入队。因此**循环队列中向后移动都用这种写法**

```c
// 初始化
queue *sq = malloc(sizeof(queue));	// 后续判空
sq->head = sq->tail = 0;	
    
// 入队,先移动索引再入队。要先判满，即 tail 下一个位置是 head
sq->tail = (sq->tail + 1) % MAXSIZE;
sq->data[sq->tail] = data;

// 出队,先移动索引再出队。要先判空，即 tail == head 
sq->head = (sq->head + 1) % MAXSIZE;
data = sq->data[sq->head];

// 遍历,从head+1开始
int i = (sq->head + 1) % MAXSIZE;
while(i != sq->tail)
{
    printf(sq->data[i]);
    i = (i + 1) % MAXSIZE;
}
printf(sq->data[i]);	// 还差tail上的没输出，因此额外输出一次

// 释放
free(sq);
```

# 链式存储队列

## 说明

把头节点后一个节点（第一个有效节点）当作队头，尾节点当作队尾。只操第一个有效节点和尾节点，因此为了提高效率，**需要额外维护一个尾指针**（双向循环不用）。

## 实现

- 初始化队列：创建头节点，即初始化链表
- 入队：在尾节点后面插入新节点，尾指针对应移动
- 出队：删除第一个有效节点，若删后为空，尾指针必须重置为 `head`
- 释放：`free`整个链表
