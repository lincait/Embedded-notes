# 进程间通信方法

进程之间的内存是隔离的，如果多个进程之间需要进行信息交换，有以下几种常用方法：Unix Domain Socket IPC、管道（有名管道、无名管道）、共享内存、消息队列、信号量。

# 管道

## 匿名管道Pipe

### 系统调用原型

`int pipe(int pipefd[2])`

`pipefd[0]`（读端）：从这个文件描述符里读取数据。

`pipefd[1]`（写端）：向这个文件描述符写入数据。

### 功能机制

- 匿名管道是内核内存中一块由环形缓冲区管理的临时存储区域。它没有名字没有文件路径，只以一对用于读写的文件描述符的形式存在。由于没有名字所以只能用于有亲缘关系的进程。
- 普通文件描述符需要通过`open`先找到或创建`struct file`，然后返回一个`fd`。但是管道在创建时就直接创建了`struct file`并返回了一对`fd`，所以管道不需要也不能`open`。
- 使用时，在`fork`之前创建管道（即一对文件描述符），`fork`后子进程也有了一对，并且和父进程指向同一个`struct file`。所以可以在一个进程写数据到写端，另一个进程从读端读数据。
- 写数据的那个进程必须要立即关闭读端，并且写完要关闭写端，这是为了告诉内核这一端用完了，不会有数据进来了，不用等了，从而触发另一进程读端的EOF。读数据的那个进程同理相反。
- 管道是单向的，一旦确定了某一方是发，那这一端在这根管道上永远只能发，否则会出现通信混乱。

## 有名管道FIFO

### 系统调用原型

`int mkfifo(char *path, mode_t)`

参数1为管道绑定的文件路径，参数2为管道绑定文件的权限

### 功能机制

- 有名管道能让任意两个进程互相通信，它绑定一个文件名，但是数据依然是从内核的环形缓冲区走。
- 想要用有名管道读写，都需要`open`获得`fd`，两个进程都获取到这个管道的`fd`时就能进行通信了。
- 管道只需要在其中一个进程创建一次，并且要在这个进程最后用`unlink()`释放管道。
- 有名管道也是单向的，所以写数据的进程`open`时给只写权限，另一进程给只读权限。
