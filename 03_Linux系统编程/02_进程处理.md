# 进程概念

## 基本概念

- 进程是正在运行的程序，是操作系统进行资源分配和调度的基本单位。

- 程序是存储在硬盘或内存的一段二进制序列，是静态的，而进程是动态的。


- 进程包括代码、数据以及分配给它的其他系统资源（如文件描述符、网络连接等）。

## 相关概念

- 进程树：Linux的进程是通过父子关系组织起来的，所有进程之间的父子关系共同构成了**进程树**。类似树状结构一样，父节点就是父进程，子节点就是子进程。

- 僵尸进程：父进程启动了一个子进程后需要在子进程结束后回收状态，如果没有正常回收，它就会在内核里一直占着位置，就会变成**僵尸进程**。

- 孤儿进程：子进程还在运行时父进程已经结束了，此时这个子进程就成为**孤儿进程**，它就会被祖先进程给领养（systemd）。

- 命令：可以用`ps -ef`命令查看所有进程。


# 相关库函数

`int system(const char *)`

使用linux命令创建一个子进程。参数为命令；成功返回0，失败返回失败编号。

# 相关系统调用

## fork

### 原型

`__pid_t fork(void)`。

`pit_t`其实是整型，用来表示进程号，发生错误时返回-1。

`getpid()`可以用来获取进程自己的进程号，`getppid()`可以获取父进程的进程号。

`sleep()`用于指定进程睡眠的秒数

### 功能机制

- 把当前进程（父进程）的全部内容复制一份，生成一个一样的进程（子进程）。生成后，父进程和子进程分别从`fork`的下一行开始运行。

- `fork`每调用一次，就会在父进程和子进程中分别返回一次。在父进程中返回**子进程的pid**，在子进程中返回0。也就是每次调用都会获得到两个值。
- `fork`后会把父进程的代码段、数据段、堆、栈全部复制给子进程，所以程序运行时父子进程虽然是独立的，但是并发运行着两个同样的代码。代码相同，但是在各自进程视角里`fork`返回的pid又是不同的，所以会运行不同的结果。
- **写时拷贝：**刚`fork`完并不会直接复制所有内容，而是让子进程指向父进程的内存，并标记为只读，只有当父子进程中任意一方去修改某个内容时，内核才会在这是复制一个新的给另一方。
- **文件共享：**父进程打开的文件描述符和文件描述符表会给子进程复制一份，但两个文件描述符里的元素指向同一个`struct file`，因此它们共用同一个文件，所以引用计数也是共用同一个变量，父进程关闭了文件只是引用计数减一，子进程还能继续写文件。（本质上就是两个进程拿到的都是指针，指向了内核里面的同一块内容）。

## execve

###原型

`int execve(char *, char * argv[], char * envp[])`

参数1是接下来要执行的程序的路径；参数2是传递给新程序的命令数组，相当于`main`的第二个参数；参数3为传递的环境变量数组。

传入的第2和3参数末尾必须明确写`NULL`。

### 功能机制

- 该函数会把当前进程运行的程序的代码、数据、堆栈抛弃，原地换成一个新的程序，但进程号不会变。
- 该函数调用成功后，已经变成了新的程序，没有旧的代码了，所以永远不会返回。失败会返回-1。
- 常常和`fork`配合，在`fork`之后让父进程仍然运行当前程序，而让子进程去执行新的程序。

## waitpid

### 原型

`pid_t waitpid(pid_t, int *, int)`

参数1为要等待的子进程ID；参数2用于接收子进程的退出状态；控制父进程的行为（阻塞或非阻塞）。

### 功能机制

- 为了防止子进程变成僵尸进程占据资源，需要`waitpid`来等待指定子进程运行完并回收它。
- `waitpid`获取子进程的退出状态后，内核就会确认“父进程已经知道子进程结束了”，然后删除并释放子进程所占的进程表槽位。
- 要等待的pid值如果传入-1，则变成等待任意子进程。
- 获取子进程状态后，要用宏解析（如`WIFEXITED`）。
- 常常在`fork`+`execve`场景下，子进程去执行了其他程序，所以必须在父进程的`if`分支里执行`waitpid`来等待子进程结束。
