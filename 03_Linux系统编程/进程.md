# 概念

进程是正在运行的程序，是操作系统进行资源分配和调度的基本单位。

程序是存储在硬盘或内存的一段二进制序列，是静态的，而进程是动态的。

进程包括代码、数据以及分配给它的其他系统资源（如文件描述符、网络连接等）。

可以用`ps -ef`命令查看所有进程。

# 相关库函数

`int system(const char *)`

使用linux命令创建一个子进程。参数为命令；成功返回0，失败返回失败编号。

# 相关系统调用

## fork

原型：`__pid_t fork(void)`。

`pit_t`其实是整型，用来表示进程号，发生错误时返回-1。

`getpid()`可以用来获取进程自己的进程号，`getppid()`可以获取父进程的进程号。

### 功能机制：

- 把当前进程（父进程）的全部内容复制一份，生成一个一样的进程（子进程）。生成后，父进程和子进程分别从`fork`的下一行开始运行。

- `fork`每调用一次，就会在父进程和子进程中分别返回一次。在父进程中返回**子进程的pid**，在子进程中返回0。也就是每次调用都会获得到两个值。
- `fork`后会把父进程的代码段、数据段、堆、栈全部复制给子进程，所以程序运行时父子进程虽然是独立的，但是并发运行着同样的代码。代码相同，但是在各自进程视角里`fork`返回的pid又是不同，所以会运行不同的结果。
- 写时拷贝（COW）：刚`fork`完并不会直接复制所有内容，而是让子进程指向父进程的内存，并标记为只读，只有当父子进程中任意一方去修改某个内容时，内核才会在这是复制一个新的给另一方。
- 父进程打开的文件描述符是共享的，并且它的文件偏移量也是共享的，因此可以多个进程写同一个文件。
