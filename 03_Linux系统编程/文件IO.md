# C标准IO库函数

以下介绍一些常见函数，还有`fwrite`、`fread`等等很多库函数。

## 打开/关闭文件

- `FILE *fopen(char *, char *)`：

  参数1为要打开的文件；参数2为打开模式，分为只读、只写、只追加写等。其中，只读模式若文件不存在会返回空且不创建文件。

  返回值为结构体指针`FILE *`，即文件。

- `int fclose(FILE *)`：

  参数为需要关闭的文件，成功返回0，失败返回EOF（负数）。

## 向文件写入数据

- `int fputc(int, FILE *)`：

  参数1为写入的数据（一个字节）；参数2为写入的文件，写在哪取决于文件打开模式。

  成功返回写入的值，失败返回EOF。

- `int fputs(char *, FILE *)`：

  参数1为需要写入的字符串，参数2为写入文件。写入成功返回非负整数。

- `int fprintf(FILE *, char *, ...)`：

  参数1为写入文件；参数2为写入的字符串（可以带格式）；可变参数3为格式对应的变量。

  成功返回写入字符的个数。

## 从文件读取数据

- `int fgetc(FILE *)`：

  参数为要读取的文件。返回读到的一个字节，出错或到文件末尾返回EOF。

  需要定义一个整型数据来接收，要输出它时可以转换成字符型。

- `char *fgets(char *, int, FILE *)`：

  参数1用于存放读取的数据；参数2为要读取的长度；参数3为读取的文件

  成功返回字符串，失败返回空。

- `int fscanf(FILE *, char *, ...)`

  参数1为读取文件；参数2为格式占位符，用于接收读取的数据；可变参数3用于存放接收的数据

  返回成功匹配到的数据个数（为格式占位符匹配到的个数）

## 标准输入输出和错误

- `stdin`：`FILE *`类型的标准输入流指针，指向终端的输入，fd为0。终端的输入会流向它
- `stdout`：`FILE *`类型的标准输出流指针，指向屏幕，fd为1。它的数据会流向屏幕输出
- `stderr`：和`stdout`一样都会把数据流到屏幕输出，fd为2。

调试报错时可以用`perror`查看，系统会自己打印错误信息。

# 系统调用

系统调用是操作系统内核提供给应用程序的API，使其可以间接访问硬件资源。

常用系统调用（`ssize_t`为有符号整数，`size_t`为无符号整数）：

- `int open(char *, int, ...)`：

  参数1为打开文件的路径；参数2为打开模式，需要多个选项时用按位或；可变参数为创建文件的权限（八进制）

- `ssize_t read(int, void *, size_t)`：

  参数1为要读取的文件描述符；参数2用于存放读取的数据；参数3为要读取的最大字节数。

- `ssize_t write(int, void *, size_t)`

- `int close(int)`

- `int stat(char *, struct stat *)`

  作用是获取一个已存在文件的元数据

  参数1为文件路径；参数2的指针指向元数据存放的位置

- `off_t lseek(int, off_t, int)`

  作用是显式地移动文件指针，即下一次读写的开始位置。

  参数1为文件描述符，参数2为偏移量（字节数），参数3为基准位置。

  返回当前位置距离文件开头的偏移量。

- 库函数`exit`和系统调用`_exit`

  两个都是退出进程，库函数的退出会进行清理操作，系统调用的退出不会执行任何清理。

# 文件描述符

在打开或创建一个文件时，操作系统会为其提供一个文件描述符（fd），是一个非负整数。

文件描述符本质上是当前进程私有的一张文件描述符表的数组下标。这个数组的每个元素都是一个指向`struct file`类型的指针，该结构体存在内核里，主要内容如下：

`f_count`：引用计数，用于管理文件对象的生命周期；

`f_pos_lock`：保护文件位置的互斥锁；

`f_pos`：当前文件位置；

`f_path`：文件路径；

`f_inode`：`struct inode`类型的指针，用于关联打开的文件和它的`inode`编号。所有文件都有一个唯一的`inode`编号，里面有文件的基础信息。

`f_op`：`struct file_operations`类型的指针，该结构体是一个函数指针的集合，它们指向了各个文件操作函数。

`private_data`：存储特定驱动或模块的私有数据。
