# 说明

- `shell`是一个命令行解释器，它接收应用程序/用户命令，然后调用操作系统内核。

- 一般情况下在终端输入命令系统就会立刻执行，如果需要重复执行大量命令，就可以把命令写成`shell`脚本，一次性运行。

- 脚本开头需要声明解释器位置。
- 脚本可以用`bash`或`sh`命令执行，也可以直接输入脚本执行（需要有执行权限）。

# 变量

- **说明：**

  脚本中的变量默认为字符串类型，不能进行运算。都使用`$`进行引用。

  脚本中不能随意使用空格，如果变量值有空格时要用双引号。

  使用`export`可以让其他脚本也能用该变量，相当于全局变量。

- **系统预定义变量：**

  `HOME、PWD、SHELL、USER`等都是系统预定义变量

- **自定义变量：**

  定义：变量名=变量值。使用`unset`撤销。

- **特殊变量：**

​	`$n`：n为具体数字，用于接收外部第n个参数（0表示脚本名称）。

​	`$#`：表示命令中输入参数的个数。

​	`$*`：表示命令中所有的参数，并且把它们看成一个整体。

​	`$@`：表示命令中所有的参数，并且把它们分开看待。

​	`$?`：判断上一个命令是否执行成功，成功返回0。

# 流程控制

## `if`和`case`

```shell
if [ 判断条件 ]
then
	程序
elif [ 判断条件 ]
then
	程序
else
	程序
fi
```

```shell
case $变量名 in
	"值1")
		程序
		;;		# 相当于break
     "值2")
        程序
        ;;
	 *)		# 相当于default
		默认程序
		;;
esac
```

## 循环

```shell
for (( 初始值;控制条件;变量变化 ))
do
	程序
done
```

```shell
for 变量 in 值1 值2 值3...	# 循环把值给变量
do 
	程序
done
```

```shell
while [ 条件 ]
do
	程序
done
```

# 函数

## 系统函数

`basename`：提取路径的文件名称

`dirname`：提取路径的绝对路径名称

## 自定义函数

- 必须在调用函数地方之前，先声明函数，shell 脚本是逐行运行。不会像其它语言一样先编译。
- Shell的函数没有形参，函数内部也用`$n`来表示函数接收到的参数。

- Shell用`return`的唯一职责就是返回状态码（0-255）。不加`return`就会以最后一条命令运行的状态码作为返回值。函数的返回值只能通过`$?`得到。

```shell
function 函数名()
{
	程序
	return 状态码;
}
```

# 工具

## cut

按行处理，从每一行切出想要的内容。通常配合`grep`来提取指定行和列的内容。

用法：`cut 选项 文件名`

`-d`可以指定依靠什么符号进行分割列；`-f`可以指定提取第几列。

## awk

拥有比cut更强大的功能，同样是切割处理。并且awk本身内置有各种变量，如`$0`，`$1`，NR等

用法：`awk 选项参数 '/pattern/{action}' 文件名`

`pattern`：awk查找的内容，即匹配的内容

`action`：找到匹配内容时所执行的的命令

# 正则表达式

正则表达式是一套用于描述文本的“公式”，它能够通过不同公式来匹配文本中任意内容。比如：判断一个字符串是不是邮箱、提取日志中的所有IP等等。

`grep`，`sed`，`awk`都支持通过正则表达式来进行匹配。

常用匹配：

- `.*`：匹配任意内容
- `.+`：匹配至少一个任意字符
- `[0-9]`：匹配一个任意数字
- `[a-zA-Z]`：匹配一个任意字母
- `^`：行开头   `$`：行结尾    `^$`：空行

# 其他

## 运算符

`$[运算式]`或`$((运算式子))`

## 条件判断

`test 判断条件`或`[ 判断条件 ]`，前后有空格。可以通过`echo $?`来判断运行结果。

- 比较两个整数：`-eq`，`-ne`，`-lt`，`-le`。

- 判断文件权限：`-r`，`-w`，`-x`。

## 读取控制台输入

`read`	(选项)	(参数)

选项：`-p`，`-t`。

参数：指定读取后存入的变量名
