整个编译过程分为 **预处理 → 编译 → 汇编 → 链接** 四大阶段。

#1. 预处理

处理源代码中以 `#` 开头的指令（如 `#include`、`#define`）。

- 做文本替换：把头文件内容原封不动插入代码，将宏名替换为定义值。
- 删除注释和多余空格。最终生成一个纯粹的 `.i` 或 `.ii` 源文件（纯文本）。

# 2. 编译

将高级语言翻译成汇编语言，内部分六步：**词法分析、语法分析、语义分析、中间代码生成、优化以及目标代码生成**。

最终结果就是汇编代码`.s`文件。

# 3. 汇编
汇编器将汇编代码转换为机器能识别的二进制机器码，生成**目标文件（.o 或 .obj）**。目标文件中包含了各个`section`，里面不仅有写的代码，还有给链接器看的内容，常见的核心部分如下：

- `.text`：代码段。存放编译后的机器指令，运行时被加载到内存的代码区。
- `.data`：已初始化数据段。存放已经赋初值的全局变量和静态变量，这些数据在程序启动时会被加载到内存。
- `.bss`：未初始数据段。这是一个**占位符**区域，存放未显示初始化的全局变量和静态变量。这部分不占用`.o`文件的实际空间。
- `.rodata`：只读数据段。存放常量数据。

# 4. 链接
将各个目标文件和所需的库进行链接，生成最终的**可执行文件**。

- 静态链接：链接时把需要用到的库直接装到可执行文件里面
- 动态链接：`gcc`默认使用的方式。链接时只把需要用到的库的文件名和符号信息放到可执行文件里，程序运行时再加载相关库。
- 混合链接：需要静态链接的库在编译时加选项`-lxx`，其他`stdio`这类库会默认用动态链接。这就是混合链接。

# 5. 运行

操作系统加载器将可执行文件读入内存，加载相关动态库，分配堆栈空间，程序开始逐条执行机器指令。
