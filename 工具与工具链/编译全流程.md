整个编译过程分为 **预处理 → 编译 → 汇编 → 链接** 四大阶段。

#1. 预处理

处理源代码中以 `#` 开头的指令（如 `#include`、`#define`）。

- 做文本替换：把头文件内容原封不动插入代码，将宏名替换为定义值。
- 删除注释和多余空格。最终生成一个纯粹的 `.i` 或 `.ii` 源文件（纯文本）。

# 2. 编译

将高级语言翻译成汇编语言，内部分六步：**词法分析、语法分析、语义分析、中间代码生成、优化以及目标代码生成**。

最终结果就是汇编代码`.s`文件。

# 3. 汇编
汇编器将汇编代码转换为机器能识别的**二进制机器码**，生成**目标文件（.o 或 .obj）**。

此时文件里已经有机器指令，但其中的函数地址还是占位符（因为调用了还没编译的库函数）。

# 4. 链接
将目标文件和所需的静态库（`.lib`/`.a`）打包在一起，**生成最终的可执行文件**。

# 5. 运行

操作系统加载器将可执行文件读入内存，加载相关动态库，分配堆栈空间，程序开始逐条执行机器指令。
