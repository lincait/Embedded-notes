# 堆和栈

堆：程序员**手动申请和释放**，存储动态分配的对象、数据结构。

栈：编译器**自动分配和释放**，存储局部变量、函数参数、返回地址。

**原则：谁申请谁释放**

# 内存管理常用函数

## malloc 

### 基本使用

```c
int *p = NULL;
p = malloc(sizeof(int));	
......
free(p);	//释放内存，即这块内存已经不属于程序了
p = NULL;	//通常释放完让它指向空，防止后面被直接使用
```

申请某一大小的空间（字节数），返回该空间的起始地址，失败返回`NULL`。

分配的内容未初始化。

### 实现数组

```c
int *p;
int num = 5;
p = malloc(sizof(int) * num);
```

此时`p`就相当于有5个元素的数组的首地址，可以当作可变长数组用。

### 函数内使用

要在某个函数内实现为函数外的指针申请内存，传入的实参如果是`p`，在函数内形参会再创建一个`p`，如果使用`p = malloc()`，申请的内存给形参`p`了，但是会在函数结束后自动释放掉，外面的`p`仍然没有获得内存。因此以下有两种做法：

1.传入`&p`，形参用二级指针接收，使用`*p = malloc()`。

2.传入`p`，但函数的返回值用`void *`，使用`p = malloc()`并`return p`。

如果传入的是指向结构体的指针`s`，结构体某一成员为指针`p`，则可以在函数中通过`s->p`为该成员直接申请。通过指针`s`来修改指针`p`。

**总的来说，传参指针`p`在函数内直接改变`p`本身不会把改变带到函数外，而可以改变`*p`或`p->n`。相当于值传递时，在函数直接改变值本身不会把改变带到函数外。**

## calloc

```c
int *p;
p = calloc(10, sizeof(int)); // 分配10个int，全部初始化为0
```

分配内存并将所有空间初始化为0

## realloc

```c
int *p = malloc(10 * sizeof(int));
int *tmp = realloc(p, 20 * sizeof(int));
```

为`p`指向的内存块调整大小

- 若新空间更大，原内容保留，新增加的部分**未初始化**。
- 若新空间更小，原内容被截断。

# typedef

## 作用

为已有的数据类型改名。在需要修改数据类型时使用，而不需要在下面的程序一个个改。

## 使用

`typedef 已有类型名 新名`

```c
//基本写法
typedef int INT;	//int-->INT
INT i;				//把i替换INT，即int i 

//指针
typedef int* IntPtr;    // IntPtr 是 int* 的别名
IntPtr p, q;			//p 和 q 都是 int*

//数组
typedef int ARR[6];	//int [6]-->ARR
ARR a;				//把a替换ARR，即int a[6]

//结构体
typedef struct node_st NODE;	//NODE为类型名

typedef struct		
{
    int i;
    float f;
}NODE;		//把该结构体类型名改为NODE

//函数
//把“参数为(int, int)，返回值为int”的函数指针命名为Op
typedef int (*Op)(int, int);	
Op fp;		//定义了上述类型的函数指针fp
```

核心：在typedef中把定义的变量名替换掉新名，可以看出定义的这个变量是什么类型。
