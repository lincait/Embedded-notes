# 结构体

## 作用

用于将多个不同类型的数据组合在一起，形成一个整体。它的主要作用包括数据封装、提高代码可读性和便于管理复杂数据。

## 类型描述

### 基本描述

```c
struct 结构体名
{
	数据类型 成员名1;
	数据类型 成员名2;
	......
};
```

此处定义只做描述，不能够同时初始化。可以把结构体的定义看成是构造了一种新的数据类型，类型名为`struct 结构体名`。

### 无头结构体

```c
struct
{
    int i;
    char ch;
}a={},b,c,*p;
```

这种描述方式没有类型名，必须同时在后面定义

## 嵌套定义

把`struct 结构体名`视为一种数据类型，因此在结构体定义时，可以把另一个结构体放进去。需要注意的时，`struct`加结构体名整体才是类型名。

## 定义变量及成员引用

```c
//定义结构体
struct simp_st
{
	int i;
	float f;
	char ch;
};

//定义变量并初始化
struct simp_st a = {123,13.2,'a'}；

//定义类型为struct simp_st的数组，每个元素都是一个对应的结构体
struct simp_st arr[2] = {{...},{...}};

//定义类型为struct simp_st的指针p，指向a
struct simp_st *p = &a;

a.i = 443;//成员引用：变量名.成员名
arr[0].i = 443;//成员引用
p->i = 443;//成员引用：指针->成员名
```

嵌套结构初始化时用另一个花括号赋值，成员引用时继续用`.`或`->`引出嵌套的成员。

## 结构体内存对齐

1.**起始地址对齐**：结构体变量的首地址能够被其最宽基本类型成员的大小整除（包括0）。

2.**成员对齐**：结构体每个成员相对于结构体首地址的**偏移量**，必须是该成员大小（或编译器的默认对齐大小，取两者中较小的那个）的整数倍。如果不够，编译器会在前面自动填充空字节。

3.**总大小对齐**：结构体的**总大小**必须是其最宽基本类型成员大小（或编译器的默认对齐大小）的整数倍。如果不满足，会在末尾填充字节。

## 函数传参

一般不直接传结构体，因为在结构体较大时，形参临时占用的空间较大

一般使用指针进行传参

# 共用体

## 作用

所有成员共用一块地址空间，所占空间是最大的成员所占空间。

同一时刻只能有一个成员生效。

## 类型描述

```c
union 共用体名
{
    数据类型 成员名1；
    数据类型 成员名2；
    ......
};
```

## 使用

定义方式、成员引用、函数传参等与结构体相同。

但在使用时同时只能使用一个成员。**例如：**前面已经对某个整型成员赋值了，那后面读取另一个浮点型成员时，会把存储内容按照浮点型来读取。

## 位域

在结构体中指定成员占用的具体比特位数，用于紧凑存储。

```c
union Data 
{
    struct 
    {
        unsigned int flag1 : 1;   // 最低位 bit0
        unsigned int flag2 : 1;   // bit1
        unsigned int value  : 6;   // bit2~bit7
    } bits;
    unsigned char byte;            // 同一字节的整体访问
};
```

`bits`结构体与`byte`占用相同的 1 字节内存（共用）：

- 修改`byte`会直接改变位域的值
- 修改任意位域成员也会改变`byte`

# 枚举

## 类型描述及定义

```c
//可以显式指定值，未指定的依次递增。
enum Color 
{
    RED,    // 枚举常量，默认 0
    GREEN,  // 默认 1
    BLUE    // 默认 2
};

enum Color c;	//枚举变量c的值为0
c = RED;		//赋值，本质是把RED替换成0，然后0给c
```

可以当成一系列的宏来使用，常用于状态机

## 所占空间

枚举常量不占空间，编译后被替换为对应值

枚举变量通常占4个字节（整型）
